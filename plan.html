PDF 轉 JPG 純網頁工具實作計畫

100% 客戶端轉換 · 無伺服器依賴 · 可直接部署至 GitHub Pages

🚀 架構與技術選型

  • PDF.js (Mozilla)：瀏覽器端 PDF 解析與 Canvas 繪製
  • JSZip & FileSaver：圖片打包 .zip 壓縮與自動下載
  • Vanilla JS + CSS3：輕量無框架，高效響應式 UI
  • IIFE 模組封裝：核心邏輯隔離，避免全域命名空間污染
  • TDD Test Runner：自研輕量單元測試模組 (test.html)

🔴🟢🔵 TDD 開發流程

1

Red（紅燈）：編寫測試

在 js/tests.js 撰寫驗證邏輯與封裝測試，確認執行失敗。

2

Green（綠燈）：最簡實作

在 js/converter.js 編寫相應程式碼，使單元測試全數通過。

3

Refactor（重構）

優化效能與異步處理邏輯，確保測試維持綠燈狀態。

📋 階段性任務實作計畫

# 模式 任務目標 關鍵預期產出
1 SDD 建置實作計畫與測試 Runner plan.html 與 test.html 基礎框架
2 TDD 檔案上傳限制單元測試與 validateFile 模組 阻擋多檔與非 PDF 格式之驗證邏輯通過綠燈
3 TDD ZIP 打包名稱與 Blob 封裝單元測試與 createZip 模組 多圖檔一鍵 Zip 壓縮包封裝邏輯通過綠燈
4 SDD 設計 index.html 介面與 style.css 拖曳預覽視覺風格 質感拖曳上傳卡片與響應式預覽網格佈局
5 SDD 整合 PDF.js 實現 renderPdfToJpgBlobs 轉換核心 PDF 逐頁繪製至 Canvas 並轉為 JPG Blob
6 SDD 完成 app.js UI 事件監聽與轉換/下載流程整合 即時進度條、圖檔預覽與打包下載全流程運作
7 TDD 自動化單元測試全過與人工 E2E 端到端驗收 test.html 100% 綠燈與完整實機測試

✅ 驗收條件與情境

  • 限制單檔 PDF：當使用者上傳非 PDF 檔或一次上傳超過 1 個檔案時，顯示錯誤訊息並阻止進行轉換。
  • 即時轉換與預覽：成功讀取 PDF 後，顯示分頁繪製進度，並能預覽各頁轉出的 JPG 圖片。
  • 打包下載 ZIP：點擊「下載全部 (ZIP)」按鈕時，將所有頁面圖片打包並提示下載。
  • GitHub Pages 相容：全靜態檔案，無跨域或後端依賴，部署即可直接公開使用。
  • RWD 全裝置適配：手機（320px）至桌機（1200px+）均有對應版型，護眼配色降低視覺疲勞。
